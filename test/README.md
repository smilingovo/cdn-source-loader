# 测试页面使用说明

这是一个用于测试 CDN 资源批量加载功能的交互式测试页面。

## 启动测试页面

```bash
npm run test
```

这将会：
1. 启动 Vite 开发服务器
2. 自动在浏览器中打开测试页面（通常是 http://localhost:3000）
3. 启用热更新功能，修改代码后会自动刷新

## 功能说明

### 配置选项

- **元数据 URL**: 输入要测试的 CDN 元数据 URL（例如：`https://unpkg.com/monaco-editor@0.54.0/min/?meta`）
- **并发数量**: 控制同时进行的请求数量（1-20）
- **重试次数**: 失败时的重试次数（0-10）
- **文件过滤**: 选择要加载的文件类型（全部/JS/CSS/JSON）

### 操作按钮

- **开始加载**: 开始批量加载资源
- **停止**: 停止当前加载任务
- **清空日志**: 清空所有日志和文件列表

### 实时统计

页面会实时显示：
- 总文件数
- 成功数量
- 失败数量
- 总进度百分比

### 日志输出

日志会显示：
- 加载进度（每 25% 更新一次）
- 成功加载的文件信息
- 加载失败的错误信息
- 完成状态

### 文件列表

显示所有文件的加载状态：
- 🟡 加载中
- 🟢 成功
- 🔴 失败

## 热更新

修改以下文件后，页面会自动刷新：
- `test/test.ts` - 测试逻辑
- `test/index.html` - 页面结构
- `src/**/*.ts` - 库源代码

## 注意事项

1. 首次加载可能需要一些时间，因为需要获取元数据并加载所有资源
2. 建议先用较小的包进行测试（如 `preact` 而不是 `monaco-editor`）
3. 如果遇到 CORS 错误，可能需要配置代理或使用支持 CORS 的 CDN
